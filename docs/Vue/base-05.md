## 1. v-once 指令

### 1.1 基本作用

`v-once` 用于标记元素或组件**仅渲染一次**，后续数据变化时不再更新视图。Vue 会为带有 `v-once` 的节点生成静态标记，在后续虚拟 DOM 的 diff 过程中直接跳过对比，从而减少计算开销。

**适用场景**：

| 场景类型     | 具体示例                                                   |
| ------------ | ---------------------------------------------------------- |
| 静态内容     | 静态标题、版权信息、配置项说明（如免责声明）               |
| 性能敏感区域 | 大数据列表中的固定部分（如表头）、高频渲染但内容不变的组件 |
| 第三方组件   | 若组件内部无动态逻辑，可包裹 `v-once` 避免冗余更新         |

### 1.2 使用示例

```vue
<template>
  <div class="product-card">
    <!-- 静态信息：使用 v-once 固化，后续变化不再更新 -->
    <h3 v-once>{{ product.title }}</h3>
    <p v-once>价格：¥{{ product.price }}</p>

    <!-- 动态信息：正常响应式更新 -->
    <p>库存状态：{{ stockStatus }}</p>
    <button @click="toggleStock">切换库存状态</button>
  </div>
</template>

<script>
export default {
  data() {
    return {
      product: {
        title: 'Vue.js 开发实战书籍',
        price: 99.0,
      },
      stockStatus: '有货',
    };
  },
  methods: {
    toggleStock() {
      this.stockStatus = this.stockStatus === '有货' ? '缺货' : '有货';
      // 尝试修改 product.title，视图不会更新（因为 v-once 已固化）
      this.product.title = 'New Title';
    },
  },
};
</script>
```

### 1.3 注意事项

| 注意点                  | 说明                                                                                      |
| ----------------------- | ----------------------------------------------------------------------------------------- |
| **数据更新失效风险**    | 使用 `v-once` 后，绑定的数据变化将不再触发视图更新，需确保内容确实无需变动。              |
| **与响应式系统的冲突**  | 若误将动态内容（如用户输入、实时状态）用 `v-once` 包裹，会导致界面与数据不一致。          |
| **与 `v-for` 结合使用** | 在 `v-for` 列表中使用 `v-once` 时，列表项一旦渲染后，即使数据源更新，旧项也不会重新渲染。 |

**慎用场景示例**：

```html
<ul>
  <li v-for="item in list" :key="item.id" v-once>{{ item.name }}</li>
</ul>
```

### 1.4 底层原理

- **编译时标记**：Vue 编译器会为 `v-once` 节点添加静态标记，渲染后直接缓存其虚拟 DOM 结构。
- **虚拟 DOM 优化**：后续更新时，Vue 会跳过静态节点的对比和 patch 操作，显著减少渲染耗时。

### 1.5 Vue 2 与 Vue 3 的差异

| 特性       | Vue 2                  | Vue 3                                                   |
| ---------- | ---------------------- | ------------------------------------------------------- |
| 静态优化   | 依赖 `v-once` 手动标记 | 自动静态提升（Static Hoisting），减少对 `v-once` 的依赖 |
| 组合式 API | 不支持                 | 可在 `<script setup>` 中配合使用                        |
| 兼容性     | 需显式添加指令         | 对静态内容的优化更智能                                  |

> **总结**：`v-once` 是 Vue 中轻量级的性能优化工具，通过冻结静态内容减少重复渲染。使用时需严格区分静态与动态内容，避免滥用导致数据更新异常。在 Vue 3 中，结合自动静态提升功能，其使用场景可进一步优化。

## 2. v-pre 指令

### 2.1 基本作用

`v-pre` 用于**跳过指定元素及其所有子元素的 Vue 编译过程**。当元素添加 `v-pre` 后，Vue 会将其视为静态内容，直接按原样输出到 HTML 中，不进行任何数据绑定、指令解析或表达式计算。

### 2.2 核心应用场景

#### ① 展示 Vue 模板语法（最常见用途）

在编写文档、教程或代码示例时，需要展示 Vue 的原始语法（如 `{{ message }}`、`v-if`、`@click` 等），而不希望 Vue 将其解析为动态内容。使用 `v-pre` 可确保语法以原始文本形式呈现。

```vue
<pre v-pre>
  <div>{{ message }}</div>
  <button @click="showMessage">点击</button>
</pre>
```

> **注意**：上述示例中如果包含 `@click` 等 Vue 指令，由于 `v-pre` 跳过编译，浏览器会尝试将 `@click` 视为原生属性并报错（`InvalidCharacterError`）。因此，在 `v-pre` 块中应只放置纯静态文本或非 Vue 指令的标记。
>
> 正确示例（仅包含插值语法）：
>
> ```vue
> <pre v-pre>
>   <div>{{ message }}</div>
> </pre>
> ```
>
> 输出结果：`{{ message }}`

#### ② 优化静态内容的渲染性能

对于大量无动态绑定的静态内容（如纯文本、静态 HTML 结构），Vue 编译器会消耗资源解析这些内容。使用 `v-pre` 可跳过编译流程，减少初始化渲染的开销。

```html
<div v-pre>
  <h1>关于我们</h1>
  <p>这是一段永远不会改变的静态文本，无需 Vue 编译。</p>
  <ul>
    <li>列表项 1</li>
    <li>列表项 2</li>
    <li>列表项 3</li>
  </ul>
</div>
```

Vue 会直接输出这段 HTML，不对其进行任何编译处理。

#### ③ 避免与其他模板引擎的语法冲突

当项目中同时使用 Vue 和其他模板引擎（如 Handlebars、EJS）时，可能出现语法冲突（例如 `{{ }}` 在 Handlebars 中也用于变量插值）。使用 `v-pre` 可隔离 Vue 的编译范围，确保其他模板引擎的语法正常工作。

```html
<!-- 引入 Handlebars 模板 -->
<script type="text/x-handlebars-template" v-pre>
  <div>{{name}}</div> <!-- Handlebars 语法，不会被 Vue 编译 -->
</script>
```

### 2.3 使用注意事项

- **完全跳过编译，禁用所有 Vue 功能**：`v-pre` 会禁用元素内的所有 Vue 特性，包括：
  - 数据绑定（如 `{{ message }}`）
  - 指令（如 `v-if`、`v-for`、`@click`）
  - 组件（子组件不会被渲染）

- **配合 `<pre>` 标签使用**：展示代码示例时，建议用 `<pre>` 标签包裹 `v-pre`，以保留代码格式（如换行、缩进）。

## 3. v-pre 与 v-once 的区别总结

| 指令     | 核心逻辑                   | 适用场景                                 | 是否会重新编译/更新      |
| -------- | -------------------------- | ---------------------------------------- | ------------------------ |
| `v-pre`  | 跳过编译，直接输出原始内容 | 展示模板语法、优化静态内容、避免语法冲突 | 不编译，永远不更新       |
| `v-once` | 编译一次，之后不再更新     | 静态内容性能优化（初次渲染后固化）       | 编译一次，数据变化不更新 |

两者都可用于性能优化，但机制不同：

- `v-pre` 连编译都不做，适合完全静态、与 Vue 无关的内容。
- `v-once` 会进行一次正常编译并缓存结果，适合初次渲染后就不会再变的内容。
