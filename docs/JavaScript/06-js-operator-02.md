位运算符直接操作数字的二进制位，是底层编程和性能优化的重要工具。在 JavaScript 中，位运算符将操作数转换为 32 位有符号整数（二进制补码形式）进行运算，然后返回结果。虽然日常开发中不常用，但在某些算法、图形处理、权限管理等场景下，位运算能提供极简高效的解决方案。本文将详细介绍 JavaScript 中的位运算符，并通过丰富的示例和经典算法题展示其应用，同时剖析 React 源码中位运算的巧妙运用。

---

## 1. 位运算符列表

| 运算符 | 名称         | 描述                                                         |
| ------ | ------------ | ------------------------------------------------------------ |
| `&`    | 按位与       | 两个对应位都为 1 时，结果位为 1，否则为 0                    |
| `\|`    | 按位或       | 两个对应位只要有一个为 1，结果位就为 1                        |
| `^`    | 按位异或     | 两个对应位不同时，结果位为 1；相同时为 0                      |
| `~`    | 按位非       | 对操作数的每一位取反（包括符号位），即 `~x = -(x+1)`         |
| `<<`   | 左移         | 将二进制位向左移动指定的位数，右边补 0                        |
| `>>`   | 有符号右移   | 将二进制位向右移动指定的位数，左边补符号位（正数补 0，负数补 1） |
| `>>>`  | 无符号右移   | 将二进制位向右移动指定的位数，左边补 0                        |

## 2. 位运算计算规则

在 JavaScript 中，位运算操作数会先被转换为 32 位有符号整数（ToInt32）。32 位有符号整数的范围是 -2^31 到 2^31-1（即 -2147483648 到 2147483647）。超出此范围的数字会被截断（只保留低 32 位）。数字以**二进制补码**形式存储，最高位（第 31 位）是符号位：0 表示正数，1 表示负数。

### 2.1 十进制转二进制（正数）

例如数字 5：
- 5 ÷ 2 = 2 余 1
- 2 ÷ 2 = 1 余 0
- 1 ÷ 2 = 0 余 1
从下往上读余数：101，所以 5 的二进制为 `101`。在 32 位中，前面补 0 至 32 位：
```
00000000000000000000000000000101
```

### 2.2 负数表示（补码）

负数采用补码表示：先取绝对值的二进制，然后按位取反，再加 1。

例如 -5：
1. 5 的二进制：`000...0101`
2. 按位取反：`111...1010`
3. 加 1：`111...1011`

所以 -5 的 32 位表示为：
```
11111111111111111111111111111011
```

### 2.3 位运算的步骤

进行位运算时，会按照 32 位二进制逐位计算，结果再转回十进制整数返回。下面每个运算符都会给出详细的逐位计算示例。

### 2.4 小数与特殊值的位运算规则

位运算操作数在进行运算之前，会先经历 **ToInt32** 抽象操作（根据 ECMAScript 规范）。这个转换规则对于小数、NaN、Infinity 以及超出 32 位范围的数字有特殊处理。理解这些规则对于避免意外的位运算结果至关重要。

#### 2.4.1 ToInt32 转换步骤

1. 将参数转换为数字（ToNumber）。
2. 如果转换后的值是 `NaN`、`+0`、`-0`、`+∞`、`-∞`，返回 `0`。
3. 否则，对数字执行以下操作：
   - 取整数部分：**向 0 取整**（即丢弃小数部分，例如 `3.7 → 3`，`-2.9 → -2`）。
   - 将得到的整数对 `2^32` 取模，映射到 32 位无符号整数范围 `[0, 2^32 - 1]`。
   - 再将这个无符号整数解释为 32 位有符号整数（即如果值 ≥ 2^31，则减去 2^32）。

#### 2.4.2 小数的位运算示例

- `5.7 & 3` → `5 & 3 = 1`
- `-2.3 | 1` → `-2 | 1 = -1`
- `0.5 << 1` → `0 << 1 = 0`

#### 2.4.3 特殊值的位运算示例

- `NaN & 1` → `0`
- `Infinity | 0` → `0`
- `-Infinity ^ 1` → `1`（因为 -Infinity 转为 0，0 ^ 1 = 1）

#### 2.4.4 超出 32 位范围的整数

当数字超出 32 位有符号整数范围（-2^31 到 2^31-1）时，ToInt32 会先对 `2^32` 取模，然后解释为有符号整数。

```javascript
console.log(2 ** 32 | 0);   // 0（因为 2^32 取模 2^32 = 0）
console.log(2 ** 32 + 5 | 0); // 5（(2^32+5) % 2^32 = 5）
console.log(-(2 ** 32) - 5 | 0); // -5（取模后为 -5）
```

#### 2.4.5 总结：位运算的隐式类型转换

| 输入值        | 转换过程                                         | 示例结果       |
| ------------- | ------------------------------------------------ | -------------- |
| 小数          | 向零取整，再执行 32 位转换                        | `3.9 & 1 → 1`  |
| `NaN`         | 直接转为 0                                       | `NaN ^ 1 → 1`  |
| `Infinity`    | 直接转为 0                                       | `Infinity >> 1 → 0` |
| `-Infinity`   | 直接转为 0                                       | `-Infinity | 0 → 0` |
| 超大整数       | 对 2^32 取模，再解释为有符号整数                  | `2**32 + 5 >> 0 → 5` |

## 3. 位运算符详解

### 3.1 按位与 `&`

**规则**：两位都为 1 时结果为 1，否则为 0。

**示例**：`5 & 3`

- 5 的 32 位二进制：`00000000000000000000000000000101`
- 3 的 32 位二进制：`00000000000000000000000000000011`
- 逐位比较：
  - 第 0 位（最低位）：1 & 1 = 1
  - 第 1 位：0 & 1 = 0
  - 第 2 位：1 & 0 = 0
  - 其余高位全 0 & 0 = 0
- 结果二进制：`00000000000000000000000000000001` → 十进制 1

```javascript
console.log(5 & 3); // 1
```

**规律总结**：
- **清零特定位**：`x & 0` 可将所有位清零。
- **保留特定位**：`x & mask` 可提取 `mask` 中为 1 的对应位。
- **判断奇偶**：`x & 1` 为 1 表示奇数，0 表示偶数。
- **与自身**：`x & x = x`。
- **与 0**：`x & 0 = 0`。
- **与 -1**：`x & -1 = x`（-1 的补码全 1）。

### 3.2 按位或 `|`

**规则**：两位只要有一个为 1，结果就为 1。

**示例**：`5 | 3`

- 5：`...0101`
- 3：`...0011`
- 逐位比较：
  - 第 0 位：1 | 1 = 1
  - 第 1 位：0 | 1 = 1
  - 第 2 位：1 | 0 = 1
  - 其余高位 0 | 0 = 0
- 结果：`...0111` → 7

```javascript
console.log(5 | 3); // 7
```

**规律总结**：
- **置位**：`x | mask` 可将 `mask` 中为 1 的对应位设为 1。
- **与自身**：`x | x = x`。
- **与 0**：`x | 0 = x`。
- **与 -1**：`x | -1 = -1`（-1 全 1）。

### 3.3 按位异或 `^`

**规则**：两位不同时为 1，相同时为 0。

**示例**：`5 ^ 3`

- 5：`...0101`
- 3：`...0011`
- 逐位：
  - 第 0 位：1 ^ 1 = 0
  - 第 1 位：0 ^ 1 = 1
  - 第 2 位：1 ^ 0 = 1
  - 其余高位 0 ^ 0 = 0
- 结果：`...0110` → 6

```javascript
console.log(5 ^ 3); // 6
```

**规律总结**：
- **交换变量**：`a ^= b; b ^= a; a ^= b;` 可交换两个数。
- **与自身**：`x ^ x = 0`。
- **与 0**：`x ^ 0 = x`。
- **与 -1**：`x ^ -1 = ~x`（-1 全 1，异或相当于取反）。
- **翻转特定位**：`x ^ mask` 可将 `mask` 中为 1 的对应位翻转。
- **结合律/交换律**：异或满足交换律和结合律，常用于校验、加密等。

### 3.4 按位非 `~`

**规则**：对每一位取反（包括符号位）。注意 `~x` 等价于 `-x - 1`。

**示例**：`~5`

- 5 的二进制：`...0101`
- 取反后：`...1010`
- 这是补码表示，对应的十进制是多少？由于最高位是 1，是负数。先取反加 1 得到原码：`...0110`（即 6），所以结果是 -6。

```javascript
console.log(~5); // -6
```

验证公式：`~x = -x - 1` → `-5 - 1 = -6`。

**规律总结**：
- **求负**：`~x + 1 = -x`。
- **与 -1**：`~(-1) = 0`。
- **与 0**：`~0 = -1`。
- **常与掩码配合**：`x &= ~mask` 用于清除 mask 对应位。

### 3.5 左移 `<<`

**规则**：将二进制位向左移动指定的位数，右边补 0。

**示例**：`5 << 2`

- 5 的二进制：`...0101`
- 左移 2 位：`...010100`（右边补两个 0） → 二进制 `10100`，即十进制 20。
```javascript
console.log(5 << 2); // 20 (5 * 4)
```

**左移规律总结**：
- **数学意义**：对于正整数，左移 n 位相当于乘以 2ⁿ。对于负数，由于补码表示，结果也等同于乘以 2ⁿ（考虑溢出）。
- **溢出处理**：当结果超过 32 位范围时，只保留低 32 位，然后解释为有符号整数。
- **示例**：
  ```javascript
  console.log(5 << 1);   // 10 (5 * 2)
  console.log(-5 << 1);  // -10 (-5 * 2)
  console.log(0x7fffffff << 1); // -2 (0x7fffffff = 2147483647，左移1后超过正范围，变成负数)
  ```
- **位移位数取模**：当移位位数大于等于 32 时，实际使用的位数是 `shift % 32`（即低 5 位）。例如 `5 << 33` 等价于 `5 << 1`。

### 3.6 有符号右移 `>>`

**规则**：将二进制位向右移动指定的位数，左边补符号位（正数补 0，负数补 1）。

**示例（正数）**：`10 >> 1`
- 10 二进制：`...1010`
- 右移 1 位，左边补 0：`...0101` → 5
```javascript
console.log(10 >> 1); // 5
```

**示例（负数）**：`-10 >> 1`
- -10 二进制（32 位补码）：`11111111111111111111111111110110`（可验证）
- 右移 1 位，左边补符号位 1：`11111111111111111111111111111011` → 这是 -5 的补码
```javascript
console.log(-10 >> 1); // -5
```

**有符号右移规律总结**：
- **数学意义**：对于正整数，右移 n 位相当于除以 2ⁿ 并向下取整（向 0 取整？实际是向负无穷方向）。对于负数，也是向下取整，例如 `-9 >> 1 = -5`（因为 -9/2 = -4.5，向下取整得 -5）。
- **示例**：
  ```javascript
  console.log(9 >> 1);   // 4 (9/2=4.5，向下取整)
  console.log(-9 >> 1);  // -5 (-9/2=-4.5，向下取整)
  console.log(-1 >> 1);  // -1 (-1/2=-0.5，向下取整得 -1)
  ```
- **移位位数取模**：同样适用 `shift % 32`。

### 3.7 无符号右移 `>>>`

**规则**：将二进制位向右移动指定的位数，左边补 0。对于正数，效果同 `>>`；对于负数，结果会变成很大的正数。

**示例**：`-10 >>> 1`
- -10 二进制同上
- 右移 1 位，左边补 0：`01111111111111111111111111111011` → 十进制 2147483643
```javascript
console.log(-10 >>> 1); // 2147483643
```

**无符号右移规律总结**：
- **特点**：始终在左边补 0，因此负数右移后符号位被清零，结果变为正数。
- **数学意义**：相当于将负数看作无符号 32 位整数进行右移，结果自然就是该无符号整数的值。
- **示例**：
  ```javascript
  console.log(10 >>> 1);  // 5
  console.log(-1 >>> 1);  // 2147483647 ( -1 的 32 位全是1，右移1位后为 0x7fffffff)
  console.log(-2 >>> 1);  // 2147483647? 实际是 2147483647? 计算：-2的补码右移1得0x7fffffff，也是 2147483647
  ```
- **移位位数取模**：同样适用 `shift % 32`。

## 4. 位运算的实际应用场景

### 4.1 权限控制（经典应用）

利用二进制位表示多种权限，通过位运算进行组合和检查，是最常见的位运算应用之一。

```javascript
// 定义权限掩码（用不同的二进制位表示不同权限）
const PERMISSIONS = {
  READ:   0b001, // 1
  WRITE:  0b010, // 2
  DELETE: 0b100  // 4
};

let userPermissions = 0;

// 赋予权限：按位或
userPermissions |= PERMISSIONS.READ | PERMISSIONS.WRITE; // 0b011 = 3

// 检查权限：按位与
const canRead = (userPermissions & PERMISSIONS.READ) !== 0;   // true
const canDelete = (userPermissions & PERMISSIONS.DELETE) !== 0; // false

// 移除权限：先取反再按位与
userPermissions &= ~PERMISSIONS.WRITE; // 0b001 = 1
```

### 4.2 标志位状态管理

在 React 源码内部，使用位运算管理多个上下文环境的状态。

```javascript
// 定义不同的上下文
const NoContext = 0b000; // 0
const AContext = 0b001;  // 1
const BContext = 0b010;  // 2

let curContext = NoContext;

// 进入 A 上下文
curContext |= AContext; // curContext = 1

// 检查是否在 A 上下文中
if (curContext & AContext) { // true
  console.log('在 A 上下文中');
}

// 离开 A 上下文
curContext &= ~AContext; // curContext = 0
```

### 4.3 快速算术运算

利用移位实现乘除 2 的幂次，性能优于普通算术运算。

```javascript
// 左移 1 位相当于乘以 2
console.log(5 << 1);  // 10

// 左移 2 位相当于乘以 4
console.log(5 << 2);  // 20

// 右移 1 位相当于除以 2 并取整
console.log(10 >> 1); // 5
console.log(9 >> 1);  // 4（向下取整）
```

### 4.4 状态压缩

将多个布尔值压缩到一个整数中，节省存储空间。BitsMap 就是典型应用，将大量布尔值用位图存储。

```javascript
// 存储 8 个布尔值在一个字节中
let flags = 0;
flags |= (1 << 0); // 设置第 0 位为 1
flags |= (1 << 3); // 设置第 3 位为 1

// 检查第 3 位
if (flags & (1 << 3)) {
  console.log('第 3 位已设置');
}
```

### 4.5 奇偶判断

利用最低位判断奇偶。

```javascript
function isOdd(n) {
  return (n & 1) === 1;
}
console.log(isOdd(5)); // true
console.log(isOdd(6)); // false
```

## 5. 算法应用案例

### 5.1 找出数组中唯一出现一次的数字

利用异或的性质 `a ^ a = 0` 和 `0 ^ a = a`。

```javascript
function singleNumber(nums) {
  return nums.reduce((acc, cur) => acc ^ cur, 0);
}
console.log(singleNumber([2, 2, 1])); // 1
```

### 5.2 判断一个数是否为 2 的幂

2 的幂的二进制表示只有一位是 1，`n & (n-1) === 0` 可判断。

```javascript
function isPowerOfTwo(n) {
  return n > 0 && (n & (n - 1)) === 0;
}
console.log(isPowerOfTwo(8));  // true
console.log(isPowerOfTwo(6));  // false
```

### 5.3 计算二进制中 1 的个数

利用 `n & (n - 1)` 消除最低位的 1，循环计数。

```javascript
function countBits(n) {
  let count = 0;
  while (n) {
    n &= n - 1;
    count++;
  }
  return count;
}
console.log(countBits(7)); // 3（111 有三个 1）
```

### 5.4 子集枚举（幂集）

利用二进制位表示元素选或不选，遍历所有掩码。

```javascript
function subsets(nums) {
  const n = nums.length;
  const result = [];
  for (let mask = 0; mask < (1 << n); mask++) {
    const sub = [];
    for (let i = 0; i < n; i++) {
      if (mask & (1 << i)) {
        sub.push(nums[i]);
      }
    }
    result.push(sub);
  }
  return result;
}
console.log(subsets([1, 2, 3]));
// 输出所有子集：[[], [1], [2], [1,2], [3], [1,3], [2,3], [1,2,3]]
```

### 5.5 不用加减乘除做加法

通过异或和与运算模拟加法。

```javascript
function add(a, b) {
  while (b !== 0) {
    const carry = (a & b) << 1; // 进位
    a ^= b;                     // 无进位和
    b = carry;
  }
  return a;
}
console.log(add(5, 3)); // 8
```

## 6. React 源码中的位运算应用

React 内部大量使用位运算来优化性能，尤其是在优先级管理和状态标记方面。本节将深入剖析 React 中如何使用位运算操作 **effectTag（现为 flags）** 和 **Lane 模型**，并给出具体的代码示例和执行流程。

### 6.1 effectTag（flags）的详细操作

在 React 的 Fiber 架构中，每个 Fiber 节点都有一个 `flags` 字段（以前称为 `effectTag`），它是一个 32 位整数，每一位表示一种副作用类型。例如：

```javascript
// 简化的 effectTag 定义（实际源码中数值可能不同）
const Placement =    0b0000000000010; // 2   - 表示需要插入 DOM
const Update =       0b0000000000100; // 4   - 表示需要更新属性
const Deletion =     0b0000000001000; // 8   - 表示需要删除
const ChildDeletion = 0b0000000010000; // 16 - 表示子节点需要删除
const ContentReset = 0b0000000100000; // 32 - 表示文本内容重置
const Ref =          0b0000001000000; // 64 - 表示 ref 更新
// ... 还有其他标志
```

这些标志通过位运算进行组合、检查和移除。

#### 6.1.1 合并多个标志（`|`）

当 Fiber 节点需要同时执行多个副作用时，React 会将这些标志按位或合并：

```javascript
// 假设一个节点既有插入操作，又有属性更新
const flags = Placement | Update; // 0b0110 = 6
```

这样 `flags` 同时包含了 `Placement` 和 `Update` 的信息。

#### 6.1.2 检查是否包含某个标志（`&`）

在 commit 阶段，React 需要知道当前节点具体要执行哪些操作。通过按位与检查：

```javascript
if (flags & Placement) {
  // 执行插入操作
}
if (flags & Update) {
  // 执行更新操作
}
```

例如 `flags & Placement` 的结果非零，说明 `Placement` 位被设置。

#### 6.1.3 移除某个标志（`& ~`）

当某个操作执行完毕后，需要从 `flags` 中移除对应的标志。通常 React 会在处理完该副作用后清除对应的位，避免重复处理。

```javascript
// 例如执行完 Placement 后
flags &= ~Placement; // 清除 Placement 位
```

`~Placement` 得到所有位取反（如 `0b111111111101`），然后按位与即可清零相应位。

#### 6.1.4 在 commit 阶段的典型应用

React 的 commit 阶段会深度遍历 Fiber 树，根据每个节点的 `flags` 执行不同的操作。下面是简化版的 `commitMutationEffects` 逻辑（基于 React 17/18 源码）：

```javascript
function commitMutationEffects(root, renderPriorityLevel) {
  let nextEffect = root.current.firstEffect;
  while (nextEffect) {
    const flags = nextEffect.flags;

    // 处理 ref 卸载（必须在 DOM 操作前）
    if (flags & Ref) {
      commitDetachRef(nextEffect);
    }

    // 根据主要标志分类处理
    const primaryFlags = flags & (Placement | Update | Deletion | Hydrating);
    switch (primaryFlags) {
      case Placement: {
        commitPlacement(nextEffect);
        // 清除 Placement 标志，表示已处理
        nextEffect.flags &= ~Placement;
        break;
      }
      case PlacementAndUpdate: {
        // 既有 Placement 又有 Update
        commitPlacement(nextEffect);
        nextEffect.flags &= ~Placement;

        const current = nextEffect.alternate;
        commitWork(current, nextEffect);
        break;
      }
      case Update: {
        const current = nextEffect.alternate;
        commitWork(current, nextEffect);
        break;
      }
      case Deletion: {
        commitDeletion(root, nextEffect, renderPriorityLevel);
        break;
      }
      // ... 其他情况
    }

    // 处理 ref 绑定（在 DOM 操作后）
    if (flags & Ref) {
      commitAttachRef(nextEffect);
    }

    nextEffect = nextEffect.nextEffect;
  }
}
```

注意：React 源码中实际上使用 `effectTag` 位掩码，并且会预先组合一些常用组合（如 `PlacementAndUpdate`）以优化 switch 分支。这里为了清晰，展示了直接判断的过程。

#### 6.1.5 为什么要用位运算？

- **效率高**：位运算直接操作二进制位，比多个布尔字段判断更快。
- **节省内存**：一个 32 位整数可以表示 32 种独立状态，比使用 32 个布尔属性省得多。
- **方便组合**：可以轻松地同时标记多种状态，并在后续按需处理。

### 6.2 Lane 模型中的位运算（补充）

React 18 的 Lane 模型也大量使用位运算管理优先级。每个 Lane 是一个 31 位掩码，优先级高的 Lane 对应数值小的位。

#### 6.2.1 添加任务

当新任务到来，将其对应的 Lane 合并到 `pendingLanes` 中：

```javascript
pendingLanes |= lane;
```

#### 6.2.2 检查任务是否存在

判断某个 Lane 是否在集合中：

```javascript
if (pendingLanes & lane) { ... }
```

#### 6.2.3 移除已完成的任务

任务完成后，将其从 `pendingLanes` 清除：

```javascript
pendingLanes &= ~lane;
```

#### 6.2.4 获取最高优先级的任务

使用 `lanes & -lanes` 技巧提取最低位的 1（即最高优先级任务）：

```javascript
const highestPriorityLane = lanes & -lanes;
```

因为负数的补码表示是取反加一，`-lanes` 会保留最低位的 1，其余位取反。这样与运算后就得到最低位的 1。

#### 6.2.5 饥饿处理

如果低优先级任务长时间未执行，会被标记为过期，放入 `expiredLanes`，然后在下次调度时强制以同步优先级执行：

```javascript
if (lanes & ExpiredLane) {
  // 强制同步执行
}
```

---

## 7. 注意事项

- 位运算仅对 32 位整数有效，超过 32 位的数字会被截断，例如 `2 ** 32` 左移可能会得到 0。
- 位运算的性能通常优于算术运算，但在现代 JavaScript 引擎中差异已不明显。
- 位运算代码可读性较低，建议在明确需要优化或算法题中使用，日常业务逻辑慎用。
- 无符号右移 `>>>` 在处理负数时会产生大正数，需要特别注意。
- React 源码中的位运算主要用于性能敏感的核心模块，如调度器和协调器，理解这些有助于深入理解 React 工作原理。
