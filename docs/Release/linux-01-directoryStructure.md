## 1、根目录/

- **作用**：文件系统的最顶层，所有目录都从这里派生。
- **示例**：

```plain
/bin
/etc
/home
/usr
...
```

---

## 2、主要目录及功能

### 2.1、 /bin

- **作用**：存放系统启动和运行所需的**核心可执行文件/命令**（对所有用户通用）。
- **示例**：

```bash
ls, cp, mv, cat, bash 等
```

### 2.2、/boot

- **作用**：存放系统启动时需要的文件，如内核（vmlinuz-\*）、引导程序（如 grub）的配置文件。
- **示例**：

```bash
vmlinuz-5.4.0-42-generic (内核文件)
initrd.img-5.4.0-42-generic (初始 RAM 磁盘)
```

### 2.3、 /dev

- **作用**：存放设备文件（虚拟或物理设备的接口），如硬盘、打印机、终端等。
- **示例**：

```bash
/dev/sda (硬盘)
/dev/tty1 (终端)
```

### 2.4、/etc

- **作用**：存放**系统配置文件**，包括服务配置、用户设置、网络配置等。
- **示例**：

```bash
/etc/passwd (用户账户信息)
/etc/hosts (主机名到IP的映射)
/etc/nginx/nginx.conf (Nginx 配置文件)
```

### 2.5、/home

- **作用**：普通用户的**个人主目录**，存放用户数据（如文档、图片等）。
- **结构**：

```bash
/home/user1/
/home/user2/
...
```

### 2.6、 /lib

- **作用**：存放系统运行所需的**共享库文件**（动态链接库，如.so 文件）。
- **示例**：

```bash
/lib/x86_64-linux-gnu/ (64位系统的库文件)
```

### 2.7、/media

- **作用**：**临时挂载外部可移动设备**（如 USB、光驱）的挂载点。
- **示例**：

```bash
/media/usb_drive (插入U盘后的访问路径)
```

### 2.8、/mnt

- **作用**：临时挂载文件系统的目录（如挂载硬盘分区或网络文件系统）。
- **示例**：

```bash
mount /dev/sdb1 /mnt/data  # 挂载硬盘分区到/mnt/data
```

### 2.9、/opt

- **作用**：安装**第三方软件或应用程序**的目录（非系统自带软件）。
- **示例**：

```bash
/opt/apache-jdk/ (自定义安装的 Java 开发工具包)
```

### 2.10、/proc

- **作用**：**虚拟文件系统**，提供内核和进程信息（实时生成，无实际文件）。
- **示例**：

```bash
/proc/cpuinfo (CPU 信息)
/proc/meminfo (内存信息)
/proc/<PID>/ (进程 <PID> 的详细信息)
```

### 2.11、/root

- **作用**：**超级用户（root）的主目录**，类似普通用户的/home。
- **路径**：

```bash
/root/.bashrc  # root 用户的配置文件
```

### 2.12、/run

- **作用**：存放系统运行时的**临时数据和锁文件**（如进程 ID 文件、套接字文件）。
- **示例**：

```bash
/run/systemd/ (systemd 相关进程信息)
```

### 2.13、/sbin

- **作用**：存放**系统管理员（root）使用的系统管理工具**（如 init, fdisk）。
- **示例**：

```bash
reboot, ifconfig, fsck
```

### 2.14、/srv

- **作用**：存放**服务（如 HTTP、FTP）的数据**，供特定服务使用。
- **示例**：

```bash
/srv/www/ (网站静态文件)
/srv/ftp/ (FTP 服务器文件)
```

### 2.15、/sys

- **作用**：**虚拟文件系统**，提供硬件设备和内核模块的接口信息。
- **示例**：

```bash
/sys/class/net/ (网络设备信息)
```

### 2.16、/tmp

- **作用**：存放**临时文件**，系统重启后通常会被清空。
- **示例**：

```bash
临时下载的文件、程序缓存
```

### 2.17、/usr

- **作用**：存放**用户程序和共享数据**（非系统核心文件）。
  - /usr/bin：用户常用命令（如 vim, firefox）。
  - /usr/lib：应用程序的共享库文件。
  - /usr/local：本地编译安装的软件（避免与系统软件冲突）。
  - /usr/share：文档、帮助文件、图标等。
  - /usr/src：内核源代码和补丁。

### 2.18、/var

- **作用**：存放**经常变化的数据**（日志、缓存、数据库等）。
  - /var/log：系统日志文件（如 syslog, nginx/access.log）。
  - /var/tmp：持久化临时文件（重启后不删除）。
  - /var/www：Web 服务器的默认文档根目录（如 Apache/Nginx）。
  - /var/cache：应用程序缓存（如包管理器的缓存）。

## 3、总结：目录设计原则

1. **职责分离**：每个目录有明确用途，避免混乱。
2. **层次清晰**：从根目录/逐层展开，结构直观。
3. **可维护性**：便于系统管理和故障排查（如日志集中到/var/log）。
